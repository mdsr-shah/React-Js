Day to Day Tasks

Day 1: Git, Rebase, Merge ---
I focused on learning Git and its essential features. I explored branching and understood how it allows developers to work on different parts of a project independently. I also studied merge and rebase, gaining insight into how changes from different branches can be integrated efficiently. While rebase was initially a bit challenging, it helped me understand how to maintain a cleaner project history. In addition, I learned about cherry-picking, which enables applying specific commits from one branch to another. Overall, this session strengthened my understanding of version control and its practical use in development workflows.

Day 2: HTML Semantic, Meta Tags --- 
I worked on HTML, focusing on meta tags and semantic tags. I learned how meta tags provide essential information about a webpage, such as its description, keywords, and character encoding, which also contribute to search engine optimization. I also explored semantic tags and understood their importance in structuring web content in a clear and meaningful way. Using elements like header, footer, and section helps improve both readability and maintainability of code.

Day 3: HTML Accessibility ---
I studied HTML accessibility and its role in creating inclusive web applications. I learned how proper use of HTML elements and attributes can enhance the usability of websites for individuals with disabilities, particularly those who rely on assistive technologies like screen readers. This session highlighted the importance of writing accessible code and encouraged me to adopt best practices to ensure a better user experience for all users.
		
Day 4: Table Layout of Website ---
I explored the use of HTML tables in designing website layouts and learned how table structures organize content into rows and columns. During this session, I studied the importance of attributes such as colspan and rowspan, which allow table cells to span across multiple columns or rows for creating flexible and structured layouts. I also practiced building simple webpage layouts using tables and understood how they were traditionally used for arranging website sections before modern CSS layout techniques like Flexbox and Grid became common. This learning experience improved my understanding of content organization and table-based structure design in HTML.

Day 5: Introduction to CSS and Responsive Units ---
I learned the fundamentals of CSS and how it is used to style and design web pages. During this session, I explored different types of CSS selectors and understood how they help target specific HTML elements for styling purposes. I also studied the importance of relative and absolute units in CSS, including units such as `px`, `em`, `rem`, `%`, `vw`, and `vh`. A major focus of the session was understanding how relative units contribute to responsive web design by allowing layouts, text, and elements to adapt smoothly across different screen sizes and devices. This helped me gain practical knowledge about creating flexible and user-friendly website designs.

Day 6: CSS Grid and Flexbox ---
I studied CSS Grid and Flexbox, two powerful layout systems used in modern web development for creating responsive and well-structured designs. During this session, I learned the purpose and practical use of both techniques, including how Flexbox helps align and distribute elements efficiently in one-dimensional layouts, while CSS Grid is useful for managing complex two-dimensional layouts with rows and columns. I also implemented these concepts in my own website project, using Grid and Flexbox to organize sections, improve alignment, and enhance the responsiveness of the overall design. This practice strengthened my understanding of modern layout design and responsive webpage structuring.

Day 7: CSS Positioning, Shadows, and Psuedo Classes ---
I learned about different positioning techniques in CSS and how they are used to control the placement of elements within a webpage layout. During this session, I explored properties such as `static`, `relative`, `absolute`, `fixed`, and `sticky`, and understood how each positioning type affects the behavior and structure of webpage elements. I also studied the use of shadows in CSS, including `box-shadow`, to create depth and modern visual effects in user interface design. In addition, I applied hover effects and pseudo-classes such as `:hover` to enhance interactivity and improve the overall user experience of my website. This session helped me understand how styling and visual effects contribute to creating attractive and dynamic web interfaces.

Day 8: CSS Animations and Transitions ---
I learned the concepts of CSS animations and transitions and how they are used to create interactive and visually appealing web interfaces. During this session, I explored transition effects for smoothly changing element properties such as color, size, and position on user interaction. I also studied CSS animations using `@keyframes` and applied animation properties to create dynamic movement effects on webpage elements. As a practical implementation, I designed and developed floating skill cards around my profile section, which enhanced the visual appearance and interactivity of my portfolio website. This session improved my understanding of motion design and modern UI enhancement techniques in web development.

Day 9: Responsive Web Design with Media Queries ---
I learned how to make websites responsive so that they can adapt effectively to different screen sizes and devices. During this session, I explored the use of CSS media queries to apply different styles based on screen width and device characteristics. I also practiced using relative units such as percentages, `em`, `rem`, `vw`, and `vh` to create flexible layouts that scale properly across desktops, tablets, and mobile devices. As part of the implementation, I improved the responsiveness of my portfolio website by adjusting layouts, typography, spacing, and navigation for smaller screens. This session enhanced my understanding of modern responsive design techniques and the importance of creating user-friendly interfaces for all devices.
