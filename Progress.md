Day to Day Tasks

Day 1: Git, Rebase, Merge ---
I focused on learning Git and its essential features. I explored branching and understood how it allows developers to work on different parts of a project independently. I also studied merge and rebase, gaining insight into how changes from different branches can be integrated efficiently. While rebase was initially a bit challenging, it helped me understand how to maintain a cleaner project history. In addition, I learned about cherry-picking, which enables applying specific commits from one branch to another. Overall, this session strengthened my understanding of version control and its practical use in development workflows.

Day 2: HTML Semantic, Meta Tags --- 
I worked on HTML, focusing on meta tags and semantic tags. I learned how meta tags provide essential information about a webpage, such as its description, keywords, and character encoding, which also contribute to search engine optimization. I also explored semantic tags and understood their importance in structuring web content in a clear and meaningful way. Using elements like header, footer, and section helps improve both readability and maintainability of code.

Day 3: HTML Accessibility ---
I studied HTML accessibility and its role in creating inclusive web applications. I learned how proper use of HTML elements and attributes can enhance the usability of websites for individuals with disabilities, particularly those who rely on assistive technologies like screen readers. This session highlighted the importance of writing accessible code and encouraged me to adopt best practices to ensure a better user experience for all users.
